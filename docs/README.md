## 🧭 Порядок чтения (для того, кто строит)
1. **[MASTER_TZ.md](./MASTER_TZ.md)** — понять продукт целиком и дорожную карту.
2. По конкретному модулю → открыть его раздел в MASTER_TZ → провалиться в детали:
   - механика → [ACADEMY_SPEC.md](./ACADEMY_SPEC.md);
   - дословно → [ACADEMY_TRANSCRIPTS.md](./ACADEMY_TRANSCRIPTS.md) (нужный урок);
   - как выглядит → `academy/<id>/frames/` или [STORYBOARD_FULL.md](./STORYBOARD_FULL.md).
3. Сверить с моделью данных в `schema.prisma`.

## 🛠 Как по этому строить (регламент)
- **GPT-мост** пишет код по разделу ТЗ (порт 9223, проект AISDR).
- **Claude** проверяет: `tsc` = 0, скриншот экрана (`shot_crm.mjs`), прохождение сценария из acceptance-критериев.
- Правки **по кругу** до зелёного → следующий раздел дорожной карты.
- Демо-режим: всё работает без внешних ключей (демо-данные/SMTP/AI).

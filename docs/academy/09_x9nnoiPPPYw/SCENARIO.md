# Реверс-инжиниринг — Видео 09: «How to use Historical reports» (1:58)

Тема: исторические отчёты. Для ТЗ — §14 Reports (Historical).

## Механика (из звука)
- Historical = снимки состава воронки во времени (как данные выглядели в прошлом: неделя за неделей, месяц за месяцем). В отличие от Insight (real-time).
- **X-ось всегда time** (выбираешь интервал). НЕЛЬЗЯ выбрать кастомный date-атрибут — потому что отчёт основан на состоянии данных на КОНЕЦ каждого периода.
- Сборка: выбрать объект → метрику. Без сегментации: число записей в любой стадии в каждый момент. С **segment by deal stage** (по историческим значениям атрибута) → сколько сделок было в каждой стадии на конец каждой недели. Toggle off closed → активная воронка.
- Примеры: active workspaces/users; total revenue from paid invoices; изменение размера/состава воронки во времени.

## Требования для ТЗ
- Тип отчёта Historical: object + metric + interval (week/month) + segment by status-атрибут (по историческим снимкам).
- Хранение исторических снимков состояния записей (snapshot на конец периода) — нужен механизм истории значений (или периодический снапшот).
- X = только время (период), не date-атрибут.

## Сценарии (acceptance)
1. Historical на Deals, segment by stage, weekly → число сделок в каждой стадии на конец каждой недели.
2. Toggle off closed → только активная воронка во времени.

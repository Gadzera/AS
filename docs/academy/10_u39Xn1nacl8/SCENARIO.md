# Реверс-инжиниринг — Видео 10: «Import your data into Objects» (7:25)

Тема: импорт CSV в объекты. Для ТЗ — §3/§10 импорт.

## Механика (из звука)
- Вход: объект → **Import / Export** → **Import CSV** → выбрать файл.
- **Маппинг:** слева колонки CSV, справа атрибуты объекта; Attio авто-сопоставляет где может; можно **создать новый атрибут** прямо в маппинге (напр. «became customer date»).
- **Уникальный атрибут = дедуп:** для каждой строки Attio ищет существующую запись по значению уник-атрибута → есть совпадение = update, нет = create. Если уник-атрибут не замаплен — создаёт новую запись на КАЖДУЮ строку. Для companies обязательно маппить колонку на **domain** (иначе дубли даже при совпадении имён).
- **Форматирование:** даты/таймстемпы — ISO/EU/US; телефоны — с кодом страны; city/state/country — в одной колонке.
- **Review:** значения с проблемами помечаются оранжевыми точками; **Continue** → preview (сколько будет created/updated) → старт.
- **Прогресс:** можно уйти со страницы; отслеживание в **workspace settings → Import history**; ошибки строк — warning с сообщением при ховере.
- **Relationship при импорте:** связь (deal→company) грузится через уник-идентификатор целевого объекта (company domain); для связи на person — email, маппленный на person email. Для любого referenced-объекта маппить колонку на его уник-атрибут.
- **Required-атрибуты:** Deals требуют name, stage, owner (иначе запись не создать).
- **Несуществующая опция stage** в review → reassign на существующую ИЛИ добавить как новый stage.
- Deals-импорт: только create (record ID ещё нет), уник-атрибут не обязателен.

## Требования для ТЗ
- Импортёр CSV: upload → авто-маппинг колонок→атрибуты + ручной маппинг + создание атрибута на лету.
- Дедуп по уник-атрибуту (update/create); companies — по domain.
- Маппинг relationship по уник-идентификатору связанного объекта.
- Валидация форматов + флаги проблем + reassign/add опций (stage).
- Preview (created/updated counts) + Import history + ошибки строк.

## Сценарии (acceptance)
1. Импорт companies.csv, website→domain → существующие обновлены, новые созданы (без дублей).
2. Импорт deals.csv: required name/stage/owner; company-колонка → associated company (по domain); typo-stage → reassign.
3. Import history показывает прогресс и ошибки.
